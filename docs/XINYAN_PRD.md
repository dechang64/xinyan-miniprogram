## 1. 产品哲学 (从 user 6 条意见提炼)

### 1.1 六条意见 (user 原话)
1. **与医疗隔离, 不是为了治疗而是未来更好**
2. **与祺臻心理是不同的小程序, 虽然也要用到相似的技术栈**
3. **照镜子也是为了更好的自己**
4. **重点放在社群建设与维护**
5. **每日一经** — 从易经、清静经、道德经、黄帝内经中选取当日应景经文简释, 可做成海报分享
6. **每日一汤** — 根据体质当然身体情况推荐每日养生美颜汤, 可以分享

### 1.2 提炼的 3 个产品灵魂

#### 灵魂 1: **「滋养」而非「治疗」**
- ❌ 不说: 祛斑 / 治疗 / 治愈 / 改善 / 缓解
- ✅ 说: 滋养 / 关爱 / 陪伴 / 焕颜 / 共修 / 涵养
- 所有 UI 文案 / 海报 / 推送 / 量表解读 都要走这根线
- **自我声明**: 「心颜是陪伴型产品, 不构成任何医学建议。内容仅供日常养生参考, 个体差异请咨询专业人士。」

#### 灵魂 2: **「照镜子, 也是为了更好的自己」**
- 这是产品情感内核, 所有页面要扣这根调
- 启动页: 「镜中, 是正在成为自己的你」
- 海报底部: 「为了更好的自己」品牌副标
- 用户 ID 默认: 「心颜客」(中性 / 不标签化)
- **不卖焦虑, 不晒完美, 只陪伴用户和自己好好相处**

#### 灵魂 3: **「共修」社群 (与普通社区区别)**
- 不叫「社区」, 叫「**心颜共修堂**」
- 主题是「**一群人一起, 慢慢变好**」, 不是 UGC 内容平台
- 行为: 共修打卡 / 经文心得 / 每日一汤分享 / 早起 / 晚安 / 自我对话
- 严守: 不卖货 / 不评美丑 / 不秀身材 / 不医疗建议 / 不政治宗教极端

### 1.3 心颜 vs 祺臻心理: 区分表

| 维度 | 祺臻心理 (qi_wechat) | 心颜 (XINYAN) |
|---|---|---|
| **目标用户** | 25-35 焦虑 / 压力 / 情绪困扰的现代人 | 30-50 关注身心滋养 / 经典文化 / 自我成长的女性 (初期) |
| **核心场景** | 倾诉 / 被听见 / 心理危机干预 | 照镜子 / 经典共修 / 日常滋养 / 群体陪伴 |
| **产品调性** | 心理 (严肃 / 专业 / 温暖) | 文艺 (安静 / 涵养 / 共修) |
| **关键差异** | 19 关键词危机 + 6 角色 + 12356 | 每日一经 + 每日一汤 + 共修堂 |
| **技术栈** | 微信云开发 + CloudBase AI | **复用**微信云开发 + CloudBase AI |
| **合规风险** | 中 (心理有边界) | 低 (养生陪伴, 无医疗) |
| **商业模型** | 心理咨询师分润 + 知识付费 | 知识付费 + 实物商品 CPS + 社群付费 |
| **AppID** | wx06d5e464bb5a2f41 | **新申请** (或复用, 待定) |

> **关键**: 两个小程序**完全独立**, 心颜**不挂「祺臻」品牌**, 避免调性混淆; 但**云函数 / 数据库 / 知识库 RAG 架构 100% 复用**, 省 60% 开发量。

### 2.1 8 大竞品地图

| 竞品 | 类别 | 核心模式 | 月费/付费 | 借鉴点 | 差异化 |

### 2.2 关键洞察 (从竞品)

1. **「每日海报 + 每日一句 + 微信扫码」是验证过的病毒模型** (潮汐日帖 + 西西弗日历 + 豆瓣电影日历), 心颜可以直接套用
2. **「圈子」vs「共修」是命名差异, 实际机制类似** — 都靠「一群人一起做」提升留存; 心颜用「共修」更有精神感
3. **「每日一汤」在女性养生市场有空缺** — 过日子 / 豆果美食都偏菜谱和复杂, 没有一款把「汤」做到「日更 + 极简 + 体质推荐」
4. **易经/经典文化类产品都偏占卜** — 缺「不占卜 / 只涵养 / 每日一解读」的产品, 心颜填这个空
5. **付费意愿**: 文艺 + 调性 + 情感价值 > 工具价值 (潮汐 18 元/月, 关键不是功能, 是「这是我想用的产品」)

### 2.3 心颜的独特位置 (一句话)
> **市面上没有「既给每日一经典, 又给每日一汤品, 还让一群人共修」的小程序 — 心颜做第一个。**

## 3. 产品架构 (6 个核心 page)

### 3.1 page 地图

```
心颜 (XINYAN)
├─ 1. 首页 (index) — 今日经文 + 今日汤品 + 共修入口
├─ 2. 每日一经 (jingwen) — 经文 + 简释 + 海报生成 + 分享
├─ 3. 每日一汤 (soup) — 体质 + 季节 + 当日汤品 + 海报生成 + 分享
├─ 4. 共修堂 (community) — 打卡 / 心得 / 早安晚安 / 排行
├─ 5. 镜中 (mirror) — 自我对话 (量表 + 日签 + 自评)
├─ 6. 我的 (me) — 收藏 / 足迹 / 设置 / 关于
```

### 3.2 6 个 page 详细设计

#### **page 1: 首页 (index)** — 第一眼定调

**目的**: 一打开就感受「这就是心颜」

**布局** (从上到下):
1. **顶部 banner** (60pt): 当前日期 + 节气 + 农历
2. **今日一经卡片** (大, 280pt): 经文标题 + 一句核心 + 简释前 2 行 + 「查看全文」按钮
3. **今日一汤卡片** (中, 200pt): 汤名 + 一句功效 + 「查看做法」按钮
4. **共修今日动态** (滚动, 150pt): 3-5 条其他用户的早安 / 晚安 / 心得
5. **3 个入口按钮** (横排, 120pt): 「照镜子」「共修堂」「汤品库」
6. **底部 tab bar**: 首页 / 经文 / 汤品 / 共修堂 / 我的

**关键体验**:
- 启动动画: 一面镜子慢慢清晰, 镜中是一行字「镜中, 是正在成为自己的你」
- 字号: 大 (中老年友好), 间距: 宽 (留白感)
- 颜色: 浅米白 + 墨绿 + 暖黄, 不刺眼

#### **page 2: 每日一经 (jingwen)** — 核心 viral 功能

**目的**: 提供 365 天应景经文, 一键生成海报分享朋友圈

**布局**:
1. **顶部**: 当前日期 + 节气 + 来源 (易经 / 清静经 / 道德经 / 黄帝内经)
2. **经文大字** (60-72pt): 一句核心经文
3. **简释** (300-500 字): 通俗化解读, 跟当下季节 / 心情对应
4. **「生成海报」按钮** (主, 120pt 高, 占满): 弹出 6 种模板选
5. **历史** (底部): 滑动选日期查看往日经文
6. **收藏** (右上): 收藏到「我的经文」

**海报模板 6 种**:
- ❶ 经典 (米色 + 竖排, 最常用)
- ❷ 节气 (随节气换底色)
- ❸ 极简 (白底黑字)
- ❹ 文艺 (手写体 + 印章)
- ❺ 水墨 (山水背景)
- ❻ 现代 (彩色 + 几何)

**分享路径**:
- 一键分享微信好友 / 朋友圈
- 海报右下角: 「心颜 · 照镜子, 也是为了更好的自己」+ 小程序码
- 朋友圈图: 1080×1920px, 上半部分经文 + 下半部分小程序码

**数据源**: 365 天经文库 (需预写) + 简释 365 篇 (每月 30 篇, 首批 30 篇)

#### **page 3: 每日一汤 (soup)** — 每日个性化推荐

**目的**: 根据体质 + 季节 + 当下身体情况推荐汤品

**布局**:
1. **体质选择** (首次进入): 9 选 1 简版体质 (阳虚 / 阴虚 / 气虚 / 痰湿 / 湿热 / 血瘀 / 气郁 / 特禀 / 平和)
2. **今日推荐** (大卡): 汤名 + 适合人群 + 食材 5-7 样 + 简单做法 (3-5 步) + 功效 1 句
3. **节气 / 当季标签**: 顶部小字
4. **「生成海报」按钮**: 跟「每日一经」同模板
5. **换一换** (辅助): 推荐 3 个备选
6. **「我喝过了」** 打卡: 进共修堂

**汤品库设计**:
- 首批 30 款, 按 4 季 + 9 体质组合
- 春夏: 绿豆 / 冬瓜 / 银耳 / 莲子 / 山药 (清淡)
- 秋冬: 排骨 / 乌鸡 / 红枣 / 当归 / 黄芪 (滋补)
- 9 体质对应汤品 (1 体质 × 4 季 × 1 款 = 36 款)

**不做什么**:
- ❌ 不做电商 (不卖食材)
- ❌ 不做菜谱详情 (只 3-5 步简版)
- ❌ 不做食材代购

**数据源**: 30 款汤品库 (预写), 来源 CCTV《生活圈》+ 北京中医药大学 + 三甲医院营养科

#### **page 4: 共修堂 (community)** — 重点中的重点

**目的**: 社群是心颜的**核心壁垒**, 一切为它服务

**布局** (复用 qi_wechat community 设计 + 升级):
1. **今日共修** (顶部): 3 个每日任务
   - ❶ 读一遍今日经文 (5 分钟)
   - ❷ 准备 / 喝下今日一汤 (15 分钟)
   - ❸ 写下 1 句自我对话 (3 分钟)
2. **打卡** (主按钮): 3 个任务完成后打卡, 生成「今日心颜签」
3. **动态** (滚动): 其他用户早安 / 晚安 / 汤品 / 心得 (匿名 + 实名可选)
4. **排行榜** (tab 切换): 今日 / 本周 / 本月 / 总共修天数
5. **3 个子版块** (tab):
   - ❶ **早安** (起床后 1 句)
   - ❷ **晚安** (睡前 1 句)
   - ❸ **汤品** (做完汤发图)
6. **关注 / 粉丝 / 互相点赞**

**严守 (红线)**:
- ❌ 不允许: 医疗建议 / 产品推销 / 减肥广告 / 身体焦虑营销
- ❌ 不允许: 晒医美 / 晒药 / 晒体重 / 晒「治疗」
- ✅ 鼓励: 日常 / 汤品 / 心情 / 早安晚安 / 经文心得
- ✅ 鼓励: 互相鼓励, 互相温暖

**数据**:
- 打卡数据: 用户每日完成 3 任务, 写进 `users` 集合
- 排行榜: 按连续打卡天数排序
- 心得: 进 `community` 集合, 走云函数过滤关键词

**云函数** (复用 qi_wechat):
- `community/post` — 发动态
- `community/feed` — 拿 feed
- `community/checkin` — 打卡
- `community/report` — 举报 (复用 v6.1 Self-Critique)

#### **page 5: 镜中 (mirror)** — 自我对话 (核心情感)

**目的**: 提供「照镜子」场景, 量化 + 质化结合的自我探索

**布局**:
1. **今日自评** (1 滑块): 心情 / 精力 / 睡眠 / 皮肤 (4 个滑块, 1-10)
2. **每日一签** (底部): 抽 1 张 (跟今日经文不同, 是心颜原创)
3. **每月一报告** (隐藏): 月底生成「本月滋养报告」(不卖货, 纯回顾)
4. **自评曲线** (可视化): 30 天心情曲线
5. **3 个轻量量表** (tab 切换, 可选):
   - ❶ PHQ-9 (心情低落) — 9 题
   - ❷ GAD-7 (焦虑) — 7 题
   - ❸ DLQI (皮肤生活质量) — 10 题
6. **「给 3 个月后的自己」** (彩蛋功能): 写 1 封信, 3 个月后自动解锁

**核心情感**:
- 「镜中, 是正在成为自己的你」
- 「不是为了治疗, 是为了未来更好」
- 「你不需要变完美, 你只需要看见自己」

**隐私保护**:
- 自评数据仅自己可见
- 量表数据不上传 (只是前端展示)
- 「给 3 个月后的自己」云端加密存 (AES-256)

#### **page 6: 我的 (me)** — 个人中心

**布局**:
1. **头像 / 昵称** (顶部)
2. **共修天数** (卡片): 连续 / 总共修
3. **收藏** (3 个 tab): 经文 / 汤品 / 心得
4. **我的海报** (列表): 历史生成的海报
5. **通知设置** (1 行)
6. **关于心颜** (1 行): 团队 / 愿景 / 联系
7. **反馈与建议** (1 行): 微信群 / 邮箱
8. **隐私政策 / 用户协议** (1 行)

## 6. 商业模型 (3 层收入)

### 6.1 第 1 层: 知识付费 (6 个月后上线)
- 「心颜会员」: 99 元/年, 解锁全部经典 / 汤品 + 海报 + 共修堂高级
- 复购率假设 30% (用户共修 1 年, 自动续费)
- 6 个月目标: 1000 付费会员, 月入 8000 元

### 6.2 第 2 层: 实物商品 CPS (12 个月后上线, 严选)
- **严选原则**: 严守「滋养而非治疗」, 选品必须 100% 合规
- **优先品类**: 厨房用品 (汤锅 / 食材罐) + 文创 (抄经笔 / 桌面摆件) + 茶饮 (花草茶 / 养生茶)
- **不选**: 保健品 / 减肥药 / 医美仪器 / 化妆品 (化妆品监管条例 17/43/46/68)
- **合作模式**: 京东 / 拼多多 CPS, 分润 5-10%
- 12 个月目标: 月均 5 万 GMV, 净利 1 万

### 6.3 第 3 层: 共修堂付费 (3 个月后试点)
- 「心颜同修小组」: 9.9 元/月, 9 人小组, 每周 1 次视频共修
- 由 1 个心颜运营 + 8 个用户组成
- 每周主题: 1 个经文 + 1 个汤品 + 1 个分享
- 3 个月目标: 50 个小组, 月入 4500 元

### 6.4 收入预测 (保守)

- 守门员: v6.1 接入

#### `tizhi/index.js` (新)
- 子 action: get / set / test
- 9 体质简版 9 问 (王琦 9 体质)

#### `poster/index.js` (新)
- 输入: type (jingwen / soup / mirror), data
- 输出: 1080×1920 海报 OSS URL
- 模板: 6 种, 用 `canvas` 包 (云函数 npm)

#### `me/index.js` (改)
